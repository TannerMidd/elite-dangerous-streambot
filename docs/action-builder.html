    Variable syntax

    Arguments for the event; globals for live state.

    %edEvent_StationName%

    Trigger argument

    A snapshot carried by the event that started this action. It only exists inside an action launched by that watcher trigger.

    ~edStation~

    Persisted global

    The watcher's current live state. It works in any action, including chat commands, hotkeys, timers, and Stream Deck buttons.

    ~edEvtDocked_StationName~

    Latest-event mirror

    The field from the most recent occurrence of an event. Use it when the current action was started by something other than that event.

    Need a field that is not in the builder?

    Choose Other event field in a condition and enter the journal field name. After a real run, confirm it in Action Queues → Action History → Inspect Variables. The full variables reference explains flattening, nested fields, arrays, and every global family.

    Before going live

    Test with the same data source the action will use.

    1

    Confirm the watcher

    Make sure the standalone C# watcher action is compiled, set to precompile on application start, and running.

    2

    Cause a real event

    Trigger arguments come from a real watcher event. Testing the action by itself does not invent fields such as %edEvent_StationName%.

    3

    Inspect the run

    Open Action Queues → Action History, right-click the run, and inspect variables to see every exact name and value.
